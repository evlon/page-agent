## 接口：语言模型对话 V3

### 基本信息

| 项目 | 说明 |
| :--- | :--- |
| **接口描述** | 提供语言模型对话能力，支持多模型、深度思考与工具调用（Function Call）。 |
| **请求方式** | `POST` |
| **请求地址** | `https://jiutian.10086.cn/largemodel/moma/api/v3/chat/completions` |
| **认证方式** | Header `Authorization: Bearer {token}` |

---

### 请求参数 (Body)

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `model` | String | ✔️ | - | 模型名称，从上方列表中选择 |
| `messages` | Array\<Message\> | ✔️ | - | 对话消息列表，最后一条必须为`user`角色 |
| `temperature` | Number | ❌ | 模型默认 | 温度 (0-1.0)，越高随机性越强 |
| `top_p` | Number | ❌ | 模型默认 | 核采样 (0-1.0)，越高多样性越高 |
| `stream` | Boolean | ❌ | `false` | 是否启用流式返回 |
| `auditSwitch` | Boolean | ❌ | `true` | 是否开启内容安全审核 |
| `tools` | Array\<Tool\> | ❌ | - | 可供模型调用的工具列表（Function Calling） |

#### Message 对象

| 参数 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `role` | String | ✔️ | `system` / `user` / `assistant` |
| `content` | String | ✔️ | 对话内容 |

#### Tool & Function 对象 (用于Function Call)

```json
{
    "type": "function",
    "function": {
        "name": "weather_api",          // 工具名称（字母/数字/_-，最大64）
        "description": "获取当前天气",   // 工具能力描述
        "parameters": {                  // 使用 JSON Schema 定义参数
            "type": "object",
            "properties": {
                "city": {
                    "type": "string",
                    "description": "城市"
                }
            },
            "required": ["city"]
        }
    }
}
```

### 返回参数

#### 非流式响应 (`stream=false`)

| 参数 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | String | 请求唯一标识 |
| `model` | String | 使用的模型名称 |
| `choices` | Array\<Choice\> | 回复列表 |
| `usage` | Object | Token 消耗统计 (`prompt_tokens`, `completion_tokens`, `total_tokens`) |
| `object` | String | 固定为 `chat.completion` |

**Choice 对象：**
```json
{
    "index": 0,
    "message": {
        "role": "assistant",
        "content": "回复内容"
    },
    "finish_reason": "stop"   // stop, length, tool_calls 等
}
```

#### 流式响应 (`stream=true`)

数据以 Server-Sent Events (SSE) 格式返回，每行以 `data: ` 开头。

**普通响应块：**
```json
{
    "choices": [{
        "delta": {
            "role": "assistant",           // 仅第一个块包含
            "content": "增量内容",          // 普通回复内容
            "reasoning_content": "思考过程" // 仅深度思考模型包含
        },
        "index": 0
    }],
    "id": "xxx",
    "model": "xxx",
    "object": "chat.completion.chunk"
}
```

**最后一个块（包含usage）：**
```json
{
    "choices": [{
        "delta": {"content": ""},
        "finish_reason": "stop",
        "index": 0
    }],
    "usage": { /* token统计 */ }
}
```

**结束标识：** `data: [DONE]`

### 代码示例

#### 1. 使用 OpenAI SDK (推荐)

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://jiutian.10086.cn/largemodel/moma/api/v3",
    api_key="<your API token>",
)

response = client.chat.completions.create(
    model="jiutian-lan-comv3",
    messages=[
        {"role": "user", "content": "你好，九天"}
    ],
    stream=False,
    max_tokens=512,
)

print(response.choices[0].message.content)
```

#### 2. 使用 Requests (流式与非流式)

```python
import requests
import json

url = "https://jiutian.10086.cn/largemodel/moma/api/v3/chat/completions"
api_key = "<your API token>"

payload = {
    "model": "jiutian-lan-comv3",
    "messages": [{"role": "user", "content": "你好，九天"}],
    "stream": True   # 流式输出
}
headers = {"Authorization": f"Bearer {api_key}"}

response = requests.post(url, json=payload, headers=headers, stream=True)

for line in response.iter_lines():
    if line:
        decoded = line.decode('utf-8')
        if decoded.startswith("data:"):
            data_str = decoded[5:].strip()
            if data_str == "[DONE]":
                break
            try:
                chunk = json.loads(data_str)
                if chunk.get("choices"):
                    content = chunk["choices"][0]["delta"].get("content", "")
                    print(content, end="", flush=True)
            except json.JSONDecodeError:
                pass
```

### 深度思考模型使用要点

1. **选择模型**：使用 `jiutian-think-v3`、`deepseek-r1`、`glm-5-fp8` 或 `kimi-k2-5-thinking`
2. **流式输出**：`stream=true` 时，`delta` 中会同时包含 `reasoning_content`（思考过程）和 `content`（最终回答）
3. **非流式**：标准 `message.content` 为最终回答，思考过程不单独返回

### 工具调用 (Function Call) 流程

1. **发起请求**：在 `tools` 参数中描述可用工具
2. **模型返回**：若模型决定调用工具，`finish_reason` 为 `tool_calls`，`message.tool_calls` 中包含调用指令
3. **客户端执行**：根据指令执行本地函数
4. **二次请求**：将工具返回结果以 `tool` 角色的消息加入 `messages` 列表，再次请求获取最终回答
